## Launch Products

The active launch apparel products are four garment/logo combinations, each with three variants:

- Brand: front art only; blank black back.
- Website promo: front art plus white `smartsleeve.ai` on the back.
- Website promo QR: front art plus white `smartsleeve.ai` and a 3.25 in x 3.25 in QR code on the back.

Core keys:

- `smartsleeve-ss-tee-brand`, `smartsleeve-ss-tee`, `smartsleeve-ss-tee-promo`
- `smartsleeve-ss-tank-brand`, `smartsleeve-ss-tank`, `smartsleeve-ss-tank-promo`
- `sqts-llc-tee-brand`, `sqts-llc-tee`, `sqts-llc-tee-promo`
- `sqts-llc-tank-brand`, `sqts-llc-tank`, `sqts-llc-tank-promo`
- `smartsleeve-ss-mousepad`, `sqts-llc-mousepad`
- `smartsleeve-ss-windbreaker`, `sqts-llc-windbreaker`
- `smartsleeve-ss-fleece`, `sqts-llc-fleece`

Mousepads are one-size, one-sided black accessories using the same SmartSleeve and SQTS front artwork as the apparel.

Outerwear products use a small right-chest logo/name lockup on black full-zip blanks. They intentionally skip back art and slogans.

## Printful Sample Order Path

For immediate personal sample orders, use Printful directly:

1. In Printful, create a new product or manual order for a black tee or black tank.
2. Pick a good-quality black blank in the size you want.
3. Add front print using the matching front PNG below.
4. For Brand variants, skip the back placement entirely.
5. For Website variants, add the matching `smartsleeve.ai` back PNG.
6. For QR variants, add the matching QR back PNG.
7. Order one sample of SS tee and SS tank first, then add SQTS samples after the SS samples look right in the physical proof.

Recommended first sample set:

- SS tee Website: `smartsleeve-ss-short-front-print.png` + `ss_and_sqts_tee_back_print.png`
- SS tank Website: `smartsleeve-ss-tank-front-print.png` + `ss_and_sqts_tank_back_print.png`
- SS tee QR promo: `smartsleeve-ss-short-front-print.png` + `ss_and_sqts_tee_back_qr_print.png`
- SS tank QR promo: `smartsleeve-ss-tank-front-print.png` + `ss_and_sqts_tank_back_qr_print.png`

Use `printful-launch-manifest.json` for the exact file map if creating all variants.

## Fulfillment Setup

SmartSleeve Quantitative Trading Systems, LLC should be the merchant of record for merch orders. The static site does not process card data directly; it opens a hosted Stripe Checkout Session created by the server-side Worker in `../merch_checkout/cloudflare_worker.js`.

Recommended production flow:

1. Create a Stripe account for SmartSleeve Quantitative Trading Systems, LLC and connect the SmartSleeve business bank account.
2. Create and publish black tee/tank products in Printful under a SmartSleeve-controlled account.
3. Run `scripts/sync_printful_storefront.py` with a Printful private token to pull the published product prices and sync variant ids.
4. Commit the generated public `printful-storefront-catalog.json` only after confirming it contains no secrets; keep `printful-sync-variants.generated.toml` private.
5. Deploy the merch checkout Worker and configure the app meta tag `smartsleeve-merch-checkout-endpoint` to point at its `/checkout` route.
6. On checkout request, the backend creates a Stripe Checkout Session with product, selected shirt size, shipping, tax, and success/cancel URLs.
7. On Stripe payment success, a backend webhook submits the order to Printful with the matching published sync variant id, then stores the Stripe session id and vendor order id.
8. Customer support, refunds, chargebacks, and accounting stay under SmartSleeve; the vendor is only a fulfillment expense.

Legacy fallback: `MERCH_PROVIDER_STORE_URL` and `MERCH_PRODUCT_URLS` can still point to provider-hosted product pages for internal testing, but those should not be the primary production path if SmartSleeve is collecting customer payment and margin.

See `../merch_checkout/README.md` for Stripe, webhook, KV, and Printful setup details.

Regenerate assets with:

```bash
.venv/bin/python scripts/generate_merch_assets.py
```
